Phase 6: The Tailwind Design System
1. The Styling Architecture
The core challenge is: How does the AI (which gives us Hex codes) talk to Tailwind (which uses utility classes)?

We use a Hybrid Styling Strategy:
Structure & Layout: Controlled by static Tailwind classes (e.g., flex, grid, absolute).
Dynamic Colors: Controlled by Inline Styles or Tailwind Arbitrary Values (e.g., bg-[${color}]).
Typography: Controlled by Tailwind's scale (e.g., text-4xl, tracking-tight).

3. Dynamic Configuration Guide
When Gemini sends the JSON, here is how you map it to Tailwind.

Handling Colors (The "Arbitrary Value" Pattern)
Gemini sends a Hex code: "#FF5733".
Don't try to match it to bg-red-500.
Do use Tailwind's arbitrary value syntax for the background: className="bg-[#FF5733]" (Constructed via template literal).
Better: Pass the hex code to the style={{ color: data.primary_color }} prop, and use Tailwind for everything else (font-bold, p-4, etc.).
Handling Text Sizing (The "Clamp" Strategy)
A major issue is long headlines breaking the design.
Strategy: Do not use fixed sizes like text-6xl.
Tailwind Solution: Use arbitrary values with CSS clamp.
Class: text-[clamp(2rem,5vw,4rem)].
This ensures the font shrinks if the screen is small or grows if the banner is large (4k export).
4. Advanced Visual Effects (Copy-Paste Ready Concepts)
Add these utility combos to make designs look expensive:
The "Gradient Text" (For Headlines):
bg-clip-text text-transparent bg-gradient-to-r from-slate-900 to-slate-600 (Makes text look metallic).
The "Inner Glow" (For Containers):
ring-1 ring-inset ring-white/10 (Adds a subtle highlight inside the border).
The "Vignette" (For Backgrounds):
Add an absolute div over the background image: bg-gradient-to-t from-black/80 via-transparent to-black/80. (Darkens top and bottom so text pops).
